History

Lua was created in 1993 by Luiz Henrique de Figueredo, and Waldemar Celes who where members of the Computer Graphics TEchnology Group (Tecgraf) at the PAcific Catholic Universiy of Rio de Janeiro, Brazil.

Brazil had strong trade barrier policies for computer hardware and software from 1977 to 1992, a lot of Tecgraf's clients cound not afford to buy cusomized software from abroad, causesing Tecgraf to implement basic software tools from scrtach. This pushed the company to create thier own custome language

Installing and building lua

you can download Lua from lua.org off from their download page. In order to Build lua

  1. open up a terminal window move ot the top-level directory lua-5.4.1.
  2. Do 'make. The MakeFile will guess your platform and bild Lua for it: Lua currently supports: guess aix bsd c89 freebsd generic linux linux-readline macosx mingw posix solaris
  3. The compilation takes only a few moments and produces three files in the src directory: lua (the interpreter), luac (the compiler), and liblua.a (the library).
  4. To check that Lua has been built correctly, do "make test" after building Lua. This will run the interpreter and print its version.

To run lua scripts just open a terminal and type: lua *file_name* it should compile and run the script!

Installing Lua

Once you've build Lua type in: sudo make install in the terminal (while you're still in the lua-5.4.1. directory)

Paradigm

Lua is a multi-paradigm language that utilizes aspects of imperative programming, Object Oriented programming and Functional programming

functions are a type of data-type in Lua and are fist class objects, you can pass a function into a parameter for another function or as a return value. Everything in Lua is an object, including functions– You can assign variables to a function or a function call. Object oriented programming using the object type: Tables, in Lua they can hold multiple type of elelments, create class objects, prototypes, and inheritence with the help of metatables and metafunctions

Typing System

  • Dynamic typing

Control Structures

Lua utilize block control flow with if, else, for conditional statments statement tags and for,while, and reapte for iteration. All control functions have an explicit terminator: end. While until terminates the repeat structure. Anything between the control tag and its termination tag is the scope of the control flow.

Semantics

Lua uses dynamic scoping, everything is run by calling context–– allowing us to implment functions in any order we please since grabing varibles/functon calls are accessed through a "call stack" that traveses up until it finds the necessary pointer we want.

Orginally Lua did not support constants automatically you had to add that functionality by puting a varible into a table and setting its metatable to read-only using a meta-table. However Lua's latest update: 5.4 update: implements a "const" tag to place before your vairbles that allow you to make it a constant

Memory management is a Heap allocated strucutre rather than a machine stack. Lua's C api also leakedthat its unque stack requires you to push and pop variables off its stack.

Lua has automatic memory management,and will run garbage collector form time to time to collect all dead objects. The garbage collector diferenciates which pointers or objects are "dead" based on strong referencing and weak referencing. Weak refernced objects will eventaully be picked up by the garbage collector. Key and values within a table are considered strong referecing. You can set the "weakness" of a table in Lua by using the function method .__mode, this allows you to indicate the tables has weak keys, values, etc. in order to set the object up for garbage collection.

Impressive Extensibility

Lua utilizes an impressive exstensibilty through the use of it most primary feautre: tables. This mechanism implements the concepts of records, arrays, and recursive data types (pointers), and adds some object-oriented capability, such as methods with dynamic dispatching. Because tables are acutally assocative arrays this has allowed Lua code to be pretty portable into C language. Lua also has a feature called Fallbacks which allows the user to extend the semantics of Lua such as utilizing a Fallback to implment inheritence for Objects.
